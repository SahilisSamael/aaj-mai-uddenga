Binary Search :

Most people use the binary search method.It is one of the most popular and effective problem-solving methods available. The location of a particular name may be found using binary search in a maximum of 35 iterations if all names in the world are listed in alphabetical order. Only a collection of sorted elements may be used for binary search. It is necessary to sort a collection before doing binary search on it. The number of iterations may always be decreased when using binary search to conduct actions on a sorted set based on the value that is being searched. Take into account the following array:

In the ninth iteration, the position of element 8 will be found using linear search. Let's look at how binary search may be used to minimize the amount of iterations. Before we begin the search, we need to know where the range begins and ends. Let us refer to them as Low and High.

LOW = 0

HIGH = (n-1)

Compare the search value K to the element in the median of the lower and higher boundaries. If the value is bigger, raise the lower bound; otherwise, lower the upper bound.S

The lower bound is 0 and the upper bound is 9, as shown in the figure above. (lower bound + upper bound) / 2 = 4 is the median of the lower and higher boundaries. Here a[4] = 4. The value 4>2, which is the one you're looking for. As a result, we don't need to search for any element beyond 4 because the elements beyond it will certainly be higher than 2. As a result, we may always set the array's upper bound to the location of element 4. Now, we repeat the operation with the following values in the same array:

Flowchart :






Algorithm :

Step I : Take array a and the key which is to be searched as input from the user.

Step ii : Initialize one variable i with zero.

Step iii: Check whether variable i is less than the length of array.

Step iv: If step[iii] is true then check key is equal to the element at variable i in the array.

Step v: If step[iv] is true then return variable i which is the index of the key in the array.

Step vi: If step[iii] is false then print key is not found in the array.






Discussion :

In this code, the user may specify the size of the array and enter values that correspond to the size of the array. When the user inputs the key to be searched in the array, it returns the index of that key in the array, but the key must be present in the array. If key is not present, it returns key not found in the array. I execute the first for loop to take the array's input and the second for loop to search for the key in the array. In this code, I execute two for loops, not nested for loops, thus the time complexity is Big O(n+n), but in general, we take the biggest power of n in the time complexity, therefore the time complexity of this code is Big O(n+n) (n). The complexity of the auxiliary space is O. (1).
