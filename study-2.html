Linear Search :

On a collection of things, linear search is utilised. It is based on the approach of traversing a list from beginning to end by studying the attributes of all components encountered along the route.

Consider an N-dimensional array of integers. You must locate and publish the positions of all elements with the value x. The linear search in this case is based on the notion of matching each element from the beginning to the end of the list with the number x and then printing the location of the element if the condition is 'True'.

If you wish to find the spots where the number 7 appears in this array. To decide the locations, each element in the array from start to finish, that is, from index 1 to index 10, will be compared with number 7, to see which element matches the number 7.






Algorithm :

Step I : Start .

Step ii : Declare an array and search element as key..

Step iii: Traverse the array until the number is found.

Step iv: If the key element is found, return the index position of the array element

Step v: If the key element is not found, return -1

Step vi: Stop.






Pseudo-Code :

PROCEDURE LINEAR_SEARCH (LIST, VALUE)

FOR EACH ITEM IN THE LIST

IF SAME ITEM == VALUE

RETURN THE ITEM’S LOCATION

END IF

END FOR

END PROCEDURE

Complexity :

TIME COMPLEXITY:

The best-case time complexity of the linear search is o(1).

The worst-case time complexity of the linear search is o(n)

SPACE COMPLEXITY:

The SPACE complexity of the linear search is o(1)






Applications :

we use linear search in following things:

• For search item in the smaller array.

• For fast searching.
